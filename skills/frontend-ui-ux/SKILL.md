# Frontend UI/UX

Use this skill for general frontend design and implementation work when the user wants a polished interface but has not supplied a strict design system or mockup.

## Default role
- Improve clarity, usability, and visual quality without changing product scope.
- Preserve existing design systems and patterns when they already exist.
- Choose one clear aesthetic direction that fits the product instead of mixing styles.

## Workflow
1. Inspect the current UI, component patterns, tokens, and layout conventions first.
2. Identify the biggest UX and visual weaknesses before changing anything.
3. Implement responsive, accessible UI that fits the current stack.
4. Add meaningful states when relevant: loading, empty, error, hover, focus, disabled.
5. Verify the result on desktop and mobile.

## Rules
- Prefer existing components, tokens, and utilities over new abstractions.
- Use typography, spacing, color, and motion deliberately; avoid generic filler styling.
- Never sacrifice function or accessibility for visual flair.
- Before using a new dependency, verify that it already exists in `package.json` or surface the install command.
- Pair this skill with `high-end-visual-design` or `minimalist-ui` only when a specific aesthetic is needed.
